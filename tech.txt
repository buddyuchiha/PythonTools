Техническое задание (ТЗ)
Проект: Конфигурационный менеджер и конвертер файлов
Язык: Python 3.8+

1. Цель проекта
Разработать утилиту для работы с конфигурационными файлами разных форматов:

Чтение, парсинг и валидация данных.

Конвертация между форматами.

Сохранение в новый файл.

TXT, INI, YAML, JSON, ENV.


2. Требования к функционалу
2.1. Поддерживаемые форматы
Формат	Описание	Библиотека
TXT	Текстовый файл (ключ=значение или произвольный текст)	Встроенные средства Python
INI	Конфигурационный файл с секциями	configparser
YAML	Структурированный формат	PyYAML
JSON	Словарь в формате JSON	json
ENV	Файл переменных окружения (.env)	python-dotenv
2.2. Основные функции
Чтение файла

Принимает путь к файлу.

Определяет формат по расширению.

Парсит содержимое в словарь Python.

Сохранение в другой формат

Принимает словарь данных и целевой формат.

Сохраняет данные в новый файл.

Конвертация

Автоматическое преобразование структур данных между форматами.

Пример: .ini → .json, .env → .yaml.

Валидация данных

Проверка на корректность ключей/значений (например, запрет спецсимволов).

Обработка ошибок (например, если файл битый).

CLI-интерфейс (опционально)

Возможность запуска с аргументами:

bash
python config_converter.py input.ini output.json
3. Технические требования
3.1. Обязательные
Класс ConfigManager, инкапсулирующий логику работы с файлами.

Методы для каждого формата:

_load_txt(), _load_ini(), _load_yaml(), _load_json(), _load_env().

_save_as_json(), _save_as_yaml(), и т.д.

Обработка ошибок:

Неверный формат файла.

Отсутствие файла.

Некорректные данные.

3.2. Дополнительно (по желанию)
Логирование действий (logging).

Юнит-тесты (pytest).

Поддержка новых форматов (например, XML или TOML).

4. Входные и выходные данные
4.1. Входные данные
Файл в одном из поддерживаемых форматов.

Пример config.ini:

ini
[database]
host = localhost
port = 5432
4.2. Выходные данные
Файл в выбранном формате.

Пример config.json:

json
{
  "database": {
    "host": "localhost",
    "port": 5432
  }
}
