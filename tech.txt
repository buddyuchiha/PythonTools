"Декоратор для логирования вызовов функций (Function Call Logger)"
Цель проекта: Создать универсальный декоратор, который будет автоматически логировать начало, конец и ошибки выполнения любой декорируемой функции. Применить его для логирования работы "модуля" для имитации работы с API.

Ключевые технологии: Python, модуль logging, декораторы.

1. Функциональные требования
Часть 1: Создание декоратора
Написать декоратор log_call. Он должен:

Принимать необязательный аргумент level для указания уровня логирования (по умолчанию logging.DEBUG).

Логировать входящие аргументы функции (и позиционные, и ключевые) с уровнем DEBUG.

Логировать начало выполнения функции с уровнем INFO: "Function 'function_name' started.".

Логировать успешное завершение функции с уровнем INFO: "Function 'function_name' finished successfully in X.Xs." (добавить время выполнения).

В случае возникновения исключения внутри функции, логировать его с уровнем ERROR: "Function 'function_name' raised an exception: ExceptionType: Error message", и затем пробрасывать исключение дальше.

Использовать functools.wraps для сохранения метаданных оригинальной функции (это важно!).

Часть 2: Настройка логирования
Настроить логгер function_logger:

Вывод в консоль (StreamHandler) для уровней INFO и выше.

Вывод в файл app.log (FileHandler) для уровней DEBUG и выше.

Формат записей должен включать: Время, Уровень, Имя логгера, Сообщение.

Часть 3: Применение декоратора
Создать фиктивный класс APIClient с методами, имитирующими работу с API. Применить декоратор @log_call ко всем его методам.

get_user_data(user_id: int) -> dict: Имитирует успешный GET-запрос. Использует time.sleep для имитации задержки сети. Возвращает словарь с данными пользователя.

post_config(config_data: dict) -> bool: Имитирует POST-запрос. С вероятностью 1 к 3 должна вызывать исключение ConnectionError("Connection timeout") для имитации сбоя сети.

check_health() -> str: Всегда возвращает "OK".

Главная функция (main):

Создать экземпляр класса APIClient.

В цикле несколько раз вызвать методы get_user_data и post_config с разными аргументами.

Поймать и обработать исключение ConnectionError от post_config, но не логировать его здесь (это должен делать наш декоратор!).
