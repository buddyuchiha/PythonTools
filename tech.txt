Техническое задание: "Валидатор и Парсер Логов"
Наименование проекта: Утилита для валидации и анализа лог-файлов.

Цель: Разработать консольную утилиту на Python, использующую регулярные выражения для проверки формата строк в лог-файлах, извлечения структурированных данных и формирования отчёта.

1. Функциональные требования
Утилита должна поддерживать два режима работы, задаваемые через аргументы командной строки.

Режим 1: Валидация (validate)

На вход принимает путь к файлу с логами.

Проверяет каждую строку файла на соответствие заданному формату.

Выводит в консоль:

Общее количество строк в файле.

Количество корректных строк.

Количество некорректных строк.

Для каждой некорректной строки выводит её номер и содержимое.

Режим 2: Анализ (analyze)

На вход принимает путь к файлу с логами.

Обрабатывает только корректные строки.

Формирует и выводит в консоль отчёт, содержащий:

Общее количество обработанных (корректных) записей.

Статистику по уровням логирования: количество записей для каждого уровня (INFO, WARNING, ERROR, DEBUG и др.).

Временной диапазон записей: самая ранняя и самая поздняя дата в логах.

2. Формат лог-файла
Ожидаемый формат каждой строки:
YYYY-MM-DD HH:MM:SS,mmm [STATUS] [module] message

Где:

YYYY-MM-DD — дата (год-месяц-день).

HH:MM:SS,mmm — время (часы:минуты:секунды,миллисекунды).

[STATUS] — уровень логирования (одно слово в верхнем регистре, напр., INFO, ERROR).

[module] — название модуля или компонента (одно слово).

message — произвольный текст сообщения.
