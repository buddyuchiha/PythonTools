Техническое задание: Многопроцессорный анализатор файлов
Цель проекта:
Создать консольное приложение на Python, которое использует модуль multiprocessing для параллельного анализа текстовых файлов (подсчёт строк, слов, символов).

Функциональные требования:
Чтение файлов в нескольких процессах:

Приложение принимает список текстовых файлов (*.txt) через аргументы командной строки (например, python analyzer.py file1.txt file2.txt).

Каждый файл обрабатывается в отдельном процессе.

Анализ содержимого файлов:
Для каждого файла программа должна подсчитать:

Количество строк.

Количество слов (разделенных пробелами/переводами строк).

Количество символов (включая пробелы и переносы строк).     

Сбор и вывод статистики:

Результаты обработки каждого файла должны выводиться в консоль в формате:

text
Файл: <имя_файла>
Строк: <N>, Слов: <M>, Символов: <K>
В конце выводится общая статистика по всем файлам.

Обработка ошибок:

Если файл не существует или не является текстовым (.txt), выводить сообщение об ошибке без остановки программы.

Технические требования:
Использование multiprocessing:

Для создания процессов использовать multiprocessing.Process или Pool.

Обмен данными между процессами организовать через Queue или Manager.

Аргументы командной строки:

Использовать модуль argparse для обработки входных параметров.

Логирование:

Добавить логирование в консоль (например, через logging) с указанием, какой процесс начал/завершил работу.

Ограничение числа процессов:

Добавить аргумент --max-processes (по умолчанию = числу ядер CPU), чтобы ограничивать количество одновременно работающих процессов.

Дополнительные задания (по желанию):
Реализовать прогресс-бар (tqdm) для визуализации выполнения.

Добавить возможность рекурсивного поиска файлов в директории.

Замерять время выполнения и сравнивать с однопроцессорной версией.
